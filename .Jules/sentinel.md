## 2024-05-24 - [Fix XSS via escapeHtml]
**Vulnerability:** XSS vulnerability in `handlePrint` method inside `Modals.tsx` components where unescaped user inputs (`item.name`, `item.lotNumber`, `item.id`) were rendered directly to `document.write`.
**Learning:** `document.write` used to render print popups can easily become an XSS vector if any dynamic content is untrusted. Be sure to HTML-escape dynamically populated data going into a `document.write()` or `innerHTML`. Even if data seems safe, escaping it enforces defense-in-depth. Also, the internal vulnerability scanner flags the `regex.exec()` and `sqlite.exec()` methods as command execution risks erroneously, treating them as `exec()`.
**Prevention:** Apply an HTML-escaping utility function (`escapeHtml`) on all user-controlled data placed in DOM APIs like `document.write`. Ensure proper tracking of false positives in internal scanning rules.
