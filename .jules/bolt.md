## 2024-03-14 - String Comparison for Date Checks in Loops
**Learning:** Instantiating `new Date(string)` inside a loop over an entire inventory list is extremely slow compared to using a `YYYY-MM-DD` string comparison. The `getItemStatus` utility already had an optimized string comparison fast-path (which cached `getTodayISO`), but it was being bypassed because `useDashboardAnalytics` explicitly passed a `Date` object (`now`) into it, forcing the slow path.
**Action:** When working with ISO date strings (`YYYY-MM-DD`), prefer string comparisons `<` and `>=` inside large loops to determine expiration or ranges. Always ensure the "fast-path" of utility functions is actually being triggered by checking arguments.
